Diagrama de Comportamiento

Casos de Uso

Muestra las interacciones entre los actores externos y el sistema. Define qué hace el sistema (no cómo), ayudando a capturar requisitos funcionales desde la perspectiva del usuario.

📌 Cuándo usarlo

Al inicio del proyecto para identificar requisitos funcionales y comunicarlos a stakeholders no técnicos.

🔑 Elementos clave

Actor (muñeco), caso de uso (elipse), sistema (rectángulo), relaciones «include» y «extend».

⚠️ Error común

Confundir pasos internos del proceso con casos de uso. Un CU debe ser observable desde fuera del sistema.

Diagrama · Casos de Uso
Ejemplos Reales
E-commerce

Cliente busca producto → añade al carrito → pago → confirmación de pedido.

Sistema Bancario

Usuario consulta saldo, transfiere, paga servicios. Admin gestiona cuentas y bloqueos.

App de Salud

Paciente agenda cita «include» autenticar. Doctor actualiza historial clínico.

Diagrama de Estructura

Diagrama de Clases

Representa la estructura estática del sistema: las clases, sus atributos, métodos y las relaciones entre ellas. Es el diagrama más utilizado en el diseño orientado a objetos.

📌 Cuándo usarlo

Durante el diseño del sistema para modelar la estructura de clases antes de escribir código.

🔑 Relaciones

Herencia (flecha hueca), Composición (diamante relleno), Agregación (diamante vacío), Dependencia (flecha punteada).

📐 Visibilidad

+ público   - privado   # protegido   ~ paquete

Diagrama · Clases
Ejemplos Reales
Sistema de Biblioteca

Libro hereda de Material. Préstamo asocia Usuario con Libro (1 usuario, muchos préstamos).

Red Social

Clase Usuario con composición de Perfil. Publicación agrega Comentarios y Reacciones.

Hospital

Médico y Paciente heredan de Persona. Cita asocia ambos con fecha, sala y diagnóstico.

Diagrama de Interacción

Diagrama de Secuencia

Muestra cómo los objetos interactúan entre sí a lo largo del tiempo, describiendo el orden cronológico de los mensajes enviados. Ideal para documentar flujos de casos de uso.

📌 Cuándo usarlo

Para documentar el flujo detallado de un caso de uso o analizar llamadas entre servicios/APIs.

🔑 Elementos clave

Línea de vida (línea punteada), activación (rectángulo), mensaje sincrónico (flecha sólida), respuesta (flecha punteada).

⏱️ Tiempo

El eje vertical representa el tiempo. Los mensajes se leen de arriba a abajo en orden cronológico.

Diagrama · Secuencia — Inicio de Sesión
Ejemplos Reales
Pago en línea

Usuario → Frontend → API de Pagos → Banco → confirmación de vuelta al usuario.

Microservicios

API Gateway distribuye request → servicio de inventario → servicio de pedidos → notificaciones.

Recuperar contraseña

Usuario solicita reset → sistema genera token → envío por email → usuario redirigido a nuevo formulario.

Diagrama de Comportamiento

Diagrama de Actividades

Modela flujos de trabajo, procesos de negocio y algoritmos. Similar a un diagrama de flujo, pero con soporte para concurrencia y carriles de actividad (swimlanes) para asignar responsabilidades.

📌 Cuándo usarlo

Para modelar procesos de negocio complejos, workflows, o describir el algoritmo de un método.

🔑 Elementos clave

Inicio (círculo negro), actividad (rectángulo redondeado), decisión (rombo), fork/join (barra negra), fin (círculo con borde).

🏊 Swimlanes

Divisiones verticales u horizontales que asignan actividades a actores o sistemas responsables.

Diagrama · Actividades — Proceso de Compra
Ejemplos Reales
Onboarding de usuario

Registro → verificar email → completar perfil → tutorial → acceso al dashboard.

Aprobación de préstamo

Solicitud → evaluación crediticia → decisión (fork: aprobado/rechazado) → notificación.

CI/CD Pipeline

Push → build → tests paralelos → análisis de código → deploy a staging → aprobación → producción.

Diagrama de Comportamiento

Diagrama de Estado

Modela el comportamiento de un objeto a través del tiempo, mostrando todos los estados posibles que puede tener y las transiciones que lo llevan de un estado a otro ante un evento.

📌 Cuándo usarlo

Para objetos con comportamiento complejo dependiente de su estado: pedidos, reservas, sesiones de usuario.

🔑 Elementos clave

Estado (rectángulo redondeado), transición (flecha con etiqueta evento[guardia]/acción), estado inicial (círculo negro), estado final.

📝 Notación

Transición: evento[condición]/acción. Ejemplo: pagar[saldo_ok]/emitir_factura

Diagrama · Estado — Ciclo de Pedido
Ejemplos Reales
Sesión de usuario

Anónimo → autenticado → bloqueado (intent fallidos) → expirado → cerrado.

Semáforo

Rojo → Verde → Amarillo → Rojo. Con transiciones temporizadas y estados de emergencia.

Ticket de soporte

Abierto → Asignado → En progreso → Resuelto → Cerrado. Opcionalmente: Reabierto.

Diagrama de Estructura

Diagrama de Componentes

Representa la arquitectura lógica del software: los módulos, librerías, servicios y sus interfaces. Muestra cómo el sistema está dividido en partes reutilizables e intercambiables.

📌 Cuándo usarlo

Para documentar la arquitectura de un sistema, especialmente en microservicios o sistemas modulares.

🔑 Elementos clave

Componente (rectángulo con ícono «component»), interfaz provista (círculo), interfaz requerida (semicírculo), dependencia.

🔌 Interfaces

Puerto provisto (lollipop): lo que el componente ofrece. Puerto requerido (socket): lo que necesita de otros.

Diagrama · Componentes — Arquitectura Web
Ejemplos Reales
Aplicación Móvil

App iOS/Android → API REST → módulos: Autenticación, Catálogo, Carrito, Pagos, Notificaciones.

Plataforma SaaS

Frontend → API Gateway → microservicios independientes con colas de mensajes (Kafka/RabbitMQ).

ERP Empresarial

Módulos de contabilidad, RRHH, inventario y CRM conectados a un bus de servicios empresarial (ESB).

Diagrama de Estructura

Diagrama de Despliegue

Describe la arquitectura física del sistema: los nodos de hardware, entornos de ejecución, y cómo los artefactos de software se distribuyen entre ellos. Fundamental para planear infraestructura.

📌 Cuándo usarlo

Para planificar la infraestructura, documentar entornos de producción y comunicar decisiones de hosting al equipo.

🔑 Elementos clave

Nodo (cubo 3D), artefacto (rectángulo con «artifact»), entorno de ejecución, comunicación entre nodos.

☁️ Nube

Los nodos pueden representar VMs, contenedores Docker, pods de Kubernetes, o regiones de AWS/GCP/Azure.

Diagrama · Despliegue — Infraestructura Cloud
Ejemplos Reales
Kubernetes Cluster

Master node + worker nodes con pods distribuidos, servicios, ingress controller y volúmenes persistentes.

Multi-región

Réplicas en us-east, eu-west y ap-south con Route 53 para DNS geolocalizado y failover automático.

On-premise + Cloud híbrido

Datacenter corporativo conectado via VPN a AWS VPC con servicios críticos en ambos entornos.
